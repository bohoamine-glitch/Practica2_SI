## 9. Análisis de puertos abiertos
### Comando:
- netstat -ano
![netstat -ano](imagen_p2/image9.png)

### 1.¿Qué puertos aparecen abiertos?
- Los puertos en estado LISTENING están abiertos<br>
![puertos_abiertos](imagen_p2/imagen9_2.png)

### 2.¿Qué significa cada columna?
- Proto: Tipo de protocolo (TCP o UDP)
- Local Address: IP y puerto del equipo 192.168.60.10:445
- Foreign Address: Dirección remota 0.0.0.0.0 -> cualquier origen
- State: LISTENING → puerto abierto, ESTABLISHED → conexión activa
- PID: Identificador del proceso que usa ese puerto

### 3.¿Qué indica el PID?
- El PID (Process ID) identifica el programa que está usando ese puerto

### Explicacion:
- El comando netstat -ano permite visualizar las conexiones de red activas y los puertos abiertos en el sistema.
Se observan puertos en estado LISTENING, lo que indica que están abiertos y esperando conexiones.
Cada línea muestra el protocolo, la dirección local, la dirección remota, el estado y el identificador de proceso (PID), que permite identificar qué aplicación está utilizando ese puerto.

### Documento consultado:
[Microsoft](https://learn.microsoft.com/en-us/windows-server/administration/windows-commands/netstat)

## 10.Consulta de la tabla ARP

- arp -a
![arp -a](imagen_p2/imagen10.png)

### 1.¿Qué dirección IP aparece asociada al otro equipo?
- 192.168.60.1 -> Esto corresponde a la puerta de enlace (router), no al otro equipo.
- No aparece 192.168.60.20 (cliente) porque: <br>
-> No está encendido windows 11 porque no puedo abrir las dos al mismo porque mi equipo no puede. 

### 2.¿Qué dirección MAC tiene?
- Para la IP 192.168.60.1:, MAC: 52-54-00-12-35-00

### Explicacion: 
- La tabla ARP relaciona: <br>
-> Dirección IP → Dirección MAC <br>
- IP: 192.168.60.1
- MAC: 52-54-00-12-35-00
- Esto permite enviar datos al dispositivo correcto dentro de la red

- El comando arp -a muestra la tabla ARP del sistema, que relaciona direcciones IP con direcciones MAC.
En la salida se observa la dirección IP 192.168.60.1 asociada a la dirección MAC 52-54-00-12-35-00, correspondiente a la puerta de enlace.
No aparece el otro equipo (192.168.60.20), lo que indica que aún no se ha establecido comunicación con él.
La tabla ARP permite al sistema identificar físicamente los dispositivos dentro de la red local.

## 11. Creación de una carpeta compartida
### 1. Crear la carpeta

- New-Item -Path C:\red -ItemType Directory
![carpeta_compartida](imagen_p2/image11.png) <br>
--> Esto crea la carpeta C:\red

### 2. Compartir la carpeta

- New-SmbShare -Name red -Path C:\red <br>
--> Esto crea un recurso compartido llamado: <br>
-win-srv01\red <br>

![compartir_carpeta](imagen_p2/image11_2.png)

### 3.Comprobar los recursos compartidos

![comprobar carpetas compartidas](imagen_p2/image11_3.png)

### ¿Cómo funcionan los recursos compartidos?
- Permiten que otros equipos accedan a archivos por red
- Desde el cliente puedes acceder a esa carpeta

### Explicacion:
- Se creó una carpeta en la ruta C:\red y se compartió en la red mediante el comando New-SmbShare.
Esto permite que otros equipos puedan acceder a dicha carpeta a través de la red utilizando el protocolo SMB.
Se comprobó la creación del recurso compartido con el comando Get-SmbShare, observando que la carpeta aparece en la lista de recursos disponibles.
